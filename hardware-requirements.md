Для функционирования подсистемы на стороне сервера достаточны следующие программные и технические средства:

**Программные средства:**
- ОС Ubuntu 24.04 LTS (или новее);
- Docker Engine 26+ и Docker Compose v2 для контейнеризации сервисов;
- СУБД PostgreSQL версии не ниже 16.0 (образ `postgres:16-alpine`);
- Redis версии не ниже 7.0 (образ `redis:7-alpine`) — брокер очередей BullMQ для фоновой обработки сплатов;
- MinIO (S3-совместимое хранилище) — для хранения исходных и обработанных файлов сплатов (`apps/api` и `apps/worker` используют `@aws-sdk/client-s3`);
- Node.js версии не ниже 20 (образ `node:20-slim`) — среда исполнения API-сервера (Fastify 4) и воркера обработки (BullMQ);
- nginx (образ `nginx:alpine`) — обратный прокси, раздача статики админ-панели и публичного веб-приложения, проксирование ассетов из MinIO;
- пакетный менеджер pnpm 9 — для управления монорепозиторием;
- Prisma CLI 5 — для миграций и генерации типов БД.

**Технические средства:**
- процессор: от 2 ядер, частота от 2 ГГц (воркер выполняет валидацию, конвертацию форматов `.ply` → `.sog`/`.spz`, генерацию LOD-манифестов, рендеринг превью-изображений из облаков точек — все операции вычислительно интенсивны);
- свободная оперативная память: от 8 ГБ (воркер загружает облака точек в память для обработки; одно облако может занимать 100–500 МБ, при параллельной обработке нескольких задач требуется запас);
- свободное дисковое пространство: от 100 ГБ (PostgreSQL хранит метаданные, версии, аннотации; MinIO хранит исходные `.ply`-файлы, сконвертированные `.sog`/`.spz`, превью-изображения и LOD-манифесты — объём растёт с количеством загруженных сплатов; рекомендован SSD для быстрой отдачи ассетов).
